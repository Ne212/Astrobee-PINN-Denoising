# 🚀 Astrobee-PINN-Denoising: Physics-Informed Neural Networks for IMU Signal Recovery

This repository contains the implementation of a **Physics-Informed Denoising Network (PIDN)** designed to clean high-noise Inertial Measurement Unit (IMU) data from the **Astrobee**—NASA's free-flying robot operating on the International Space Station (ISS).

Our model effectively suppresses intense "sawtooth" thruster noise where traditional filters (Kalman, Butterworth, etc.) fall short, by leveraging physical sensor calibration layers and spectral loss constraints to achieve SOTA (State-of-the-Art) performance.

---

## 🏆 Key Achievements (SOTA Results)

On unseen test datasets, the PIDN model outperformed classical methods with the following metrics:

* **Overall RMSE Improvement:** **72.8%** reduction in error compared to raw data (peaking at **83.82%** on the Y-axis).
* **Absolute Error (RMSE) Reduction:** Lowered from $\sim 1.23 \text{ m/s}^2$ to a clean **$0.35 \text{ m/s}^2$**.
* **Signal-to-Noise Ratio (SNR) Gain:** Transformed signal quality from noise-dominant ($-2.9 \text{ dB}$) to signal-dominant (**$8.62 \text{ dB}$**).
* **Automated Calibration:** The model successfully isolated and corrected a hardware bias of $\sim 0.18 \text{ m/s}^2$ without prior manual calibration data.

## 🧠 Model Architecture (PIDN)

The architecture is designed to be "Physics-Informed" rather than a standard black-box neural network. Total parameters: **1,213,580**.

1. **Sensor Calibration Layer:** The raw IMU data first passes through a physical layer that learns the cross-axis coupling ($S$ matrix) and sensor bias ($\mathbf{b}$) to perform an initial correction ($\mathbf{a}_{cal} = S \cdot (\mathbf{a}_{raw} - \mathbf{b})$).
2. **Deep Encoder (MultiScale CNN + BiLSTM):** A 128-channel Dilated CNN extracts noise features across multiple frequency bands, followed by a 2-layer Bidirectional LSTM to solve the temporal kinematics of the movement.
3. **Residual Connection:** The network does not predict the acceleration directly; instead, it learns the correction factor ($\delta$) to be added to the physically calibrated acceleration ($\mathbf{a}_{cal}$).

---

## 🔬 Physics-Informed Loss Function (PINN Loss)

Model training is based on the simultaneous optimization of three competing physical/spectral constraints:

* **$\mathcal{L}_{accel}$ (Accuracy - Weight: 1000.0):** A massive MSE penalty ensuring the model adheres strictly to the Ground Truth trajectory.
* **$\mathcal{L}_{spectral}$ (Frequency - Weight: 25.0):** An FFT-mask-based penalty designed to crush the high-frequency sawtooth noise (>6.0 Hz) generated by Astrobee's propulsion system.
* **$\mathcal{L}_{smooth}$ (Smoothness - Weight: 0.1):** A derivative-based penalty to prevent non-physical sudden jumps (Jerk).

---

## 📊 Dataset & Training Strategy

* **Data:** 18 flight scenarios consisting of raw IMU telemetry (62.5 Hz) from the ISS.
* **Windowing:** Trained on 15,644 high-overlap windows (Train: 10,950, Val: 2,346, Test: 2,348) using `Sequence Length = 200` and `Stride = 30`.
* **Super-Convergence:** Utilized `OneCycleLR` to aggressively ramp up the Learning Rate to $\sim 1.96e-03$, allowing the model to escape local minima under the heavy $\mathcal{L}_{accel}$ constraint.
* **Early Stopping:** Training successfully converged at epoch 168 out of 400.
